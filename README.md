# Codex Configurator

这是一个专为 Codex 桌面端/CLI 设计的轻量级本地配置工具。通过交互式终端问答，一键接管并生成 Codex 的本地配置文件，将请求流量无缝路由至你指定的自定义 API 节点。

## 核心特性

* **交互式部署**：运行后根据终端提示输入信息，告别繁琐的手动改文件和环境变量注入。
* **默认拉满性能**：底层自动锁定 `gpt-5.4` 模型，并默认开启最高级别推理 (`model_reasoning_effort = "xhigh"`)，确保代码生成的逻辑严密性。
* **按需系统通知**：可选开启任务完成后的系统级弹窗通知（原生支持 macOS AppleScript 与 Windows CMD msg）。
* **纯本地透明执行**：剥离所有发卡平台与网络远程验证，你的 API Key 仅写入本地目录的 `auth.json`，确保资产安全。

## 运行环境

* 操作系统：Windows, macOS, 或 Linux
* 基础环境：请确保系统已安装 Node.js

## 快速上手

**1. 初始化环境**
在脚本所在目录下，首先安装必须的文件处理依赖：

```bash
npm install fs-extra @iarna/toml
```

**2. 执行配置**
在终端中直接运行该脚本：

```bash
node index.js
```

**3. 跟随终端指引**
系统会依次向你发起提问，请根据实际情况输入：
1. **API 代理地址**（直接回车则默认使用官方地址 `https://api.openai.com/v1`）
2. **API 密钥**（必须填写你的 Token）
3. **任务通知开关**（输入 y 开启完成弹窗，输入 n 保持静默）

看到“配置完成”的提示后，重新启动你的 Codex 客户端即可直接使用。

## 底层逻辑与维护说明

* **无损覆盖**：如果需要更换节点或密钥，无需手动清理，再次运行 `node index.js` 即可自动覆写最新配置。
* **配置隔离**：本工具的所有修改均严格限制在用户目录 `~/.codex/` 内（涉及 `config.toml` 与 `auth.json`），绝不污染系统级的全局环境变量。
