# Codex Configurator

这是一个专为 Codex 桌面端/CLI 设计的轻量级本地配置工具。它剥离了所有特定平台的远程鉴权逻辑，仅需在本地通过交互式问答，即可一键接管并生成 Codex 的本地配置文件，将请求流量路由至你指定的自定义 API 节点。

## 核心特性

* **现代交互式部署**：运行后跟随终端指引，输入你的 API 信息即可完成配置。告别繁琐的手动改文件和侵入性的环境变量设置。
* **默认性能拉满**：底层自动锁定 `gpt-5.4` 模型，并默认开启最高级别推理 (`model_reasoning_effort = "xhigh"`)，确保代码生成的逻辑严密性。
* **现代系统通知 (Windows/macOS)**：
    * **Windows**：可选配置标准的 Toast 通知，大字标题为“codex当前任务已完成”，小字正文为“codex做了什么”。
    * **macOS**：可选配置标准的 AppleScript 通知。
* **纯本地，零依懒（运行后）**：你的 API Key 和代理地址仅保存在本地的配置文件中，绝对透明安全。

## 运行环境

* 操作系统：Windows, macOS, 或 Linux
* 基础环境：请确保系统已安装 Node.js

## 快速上手

确保你的电脑装有 Node.js。下载 `index.js` 文件，在所在目录打开终端，直接运行：

```bash
node index.js
```

跟随提示输入 API 和 密钥即可，没有任何乱七八糟的安装步骤！

## 底层维护说明

* **配置隔离**：本工具的所有修改均严格限制在用户目录 `~/.codex/` 内（涉及 `config.toml` 与 `auth.json`），绝不污染系统级的全局环境变量。
* **通知脚本**：如果开启了通知，脚本会在 `~/.codex/` 下生成 `notify_on_finish.ps1` (Windows) 或 `notify_on_finish.sh` (macOS)，并在 `config.toml` 中配置 Codex 在任务完成时调用它们。
